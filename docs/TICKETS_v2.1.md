#### TICKET-2.2: Форма сообщения и отправка
**Приоритет:** 🟡 Высокий  
**Время:** 2 часа

**Требования:**
После выбора студентов — форма:

```
✉️ Сообщение для 3 студентов

Настройки:
☑️ Именное обращение ("Привет, {имя}!")

Текст сообщения:
━━━━━━━━━━━━━━━━━━━━━━━━━━━━
Внесены изменения в требования
к оформлению работ. Проверьте
документ в разделе "Мои работы".
━━━━━━━━━━━━━━━━━━━━━━━━━━━━

📎 Прикрепить файл: [📎 Прикрепить]
(макс. 1 файл для всех)

📅 Новый дедлайн (опционально):
[📅 Указать дату]

⚙️ Настройки отправки:
Задержка между сообщениями: [15] сек
(настройка для защиты от блокировки)

[👁️ Предпросмотр] [📤 Отправить]
```

**Функционал:**
1. **Именное обращение:** если включено, подставляется имя студента
2. **Прикрепление файла:** один файл для всех получателей
3. **Новый дедлайн:** если указан — обновляется у всех выбранных
4. **Throttling:** задержка между сообщениями (по умолчанию 15 сек, настраиваемый параметр)
5. **Предпросмотр:** показать, как будет выглядеть сообщение

**Throttling обязателен:**
```python
# Псевдокод
for student in selected_students:
    await send_message(student, message)
    await asyncio.sleep(throttling_seconds)  # 15 сек по умолчанию
```

**БД:**
```sql
CREATE TABLE antiplag_requests (
    id SERIAL PRIMARY KEY,
    work_id INTEGER REFERENCES student_works(id),
    student_id INTEGER REFERENCES users(id),
    request_type VARCHAR(20), -- 'primary', 'revision'
    status VARCHAR(20), -- 'pending', 'code_provided', 'confirmed', 'revision_requested'
    code VARCHAR(255),
    uniqueness_percent INTEGER,
    created_at TIMESTAMP DEFAULT NOW(),
    updated_at TIMESTAMP
);
```

### Цветовая логика статусов
```python
def get_work_status_color(work):
    if work.grade_classic or work.grade_100:  # есть оценка
        return "🟢"
    elif work.status == 'rejected':
        return "🔴"
    elif work.deadline and work.deadline < now():
        return "🔴"
    elif work.file_path:  # есть файл
        return "🟡"
    else:
        return "🔴"  # нет файла
```
