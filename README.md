# StudoBot 🎓

**Система управления заданиями для преподавателя** — персональный помощник для отслеживания работ студентов, курсовых, ВКР, научных статей и проектной деятельности.

> "Не хочу вводить списки, но хочу помнить про каждого, кто ко мне обращается"

## 🎯 Зачем это нужно?

StudoBot решает проблемы, с которыми сталкивается преподаватель:
- **Потеря контекста** — студенты пишут в разных мессенджерах, истории очищаются
- **Хаос в файлах** — документы разбросаны по почте, Telegram, WhatsApp
- **Забытые дедлайны** — нет единого календаря по всем работам
- **Повторяющиеся ответы** — одни и те же вопросы от разных студентов

## 📦 Компоненты системы

### 1. Telegram Bot
- Приём файлов и сообщений от студентов
- Генерация персональных ссылок-приглашений
- Команды для навигации по своим работам

### 2. API (FastAPI)
- REST API для всех операций
- WebSocket для real-time уведомлений
- OpenAPI документация

### 3. База данных (PostgreSQL)
- Гибкая схема без жёстких ограничений
- Миграции через Alembic
- Полнотекстовый поиск

### 4. Хранилище файлов (MinIO)
- S3-совместимое хранилище
- Версионирование файлов
- Контроль целостности (SHA256)

### 5. n8n Workflows
- Автоматические напоминания
- Массовые рассылки
- Интеграция с AI

### 6. Веб-панель
- Дашборд преподавателя
- Управление работами
- Экспорт отчётов

## 🚀 Быстрый старт

### Предварительные требования

- Docker и Docker Compose
- Telegram Bot Token (от [@BotFather](https://t.me/BotFather))
- (Опционально) API ключ для AI-модели

### Установка

```bash
# Клонирование репозитория
git clone https://github.com/your-username/studobot.git
cd studobot

# Копирование конфигурации
cp .env.example .env

# Редактирование настроек
nano .env

# Запуск
docker-compose up -d
```

### Первый запуск

1. Откройте бота в Telegram
2. Отправьте `/start` для регистрации
3. Бот создаст ваш профиль преподавателя
4. Отправьте ссылку-приглашение студентам

## 📋 Типы поддерживаемых работ

| Тип | Пайплайн | Артефакт |
|-----|----------|----------|
| **Базовое задание** | Сдача → Проверка | Файл |
| **Курсовая работа** | Тема → Структура → Черновики → Финал | Файл (Word/PDF) |
| **ВКР** | Тема → Главы → Предзащита → Финал | Файл (PDF) |
| **Научная статья** | Тема → Текст → Рецензия → Публикация | Ссылка на eLIBRARY |
| **Проект** | Сценарий → Milestone'ы → Релиз | Файл или ссылка |
| **Аспирантура** | Параллельные: 3 статьи + диссертация + эссе | Смешанный |

## 🛡️ Безопасность

- **SHA256 контрольные суммы** — доказательство сдачи в срок
- **Версионирование** — история всех изменений файлов
- **Резервное копирование** — автоматический бэкап на Яндекс.Диск
- **Изоляция данных** — каждый студент видит только свои работы

## 🔧 Конфигурация

### Переменные окружения

```bash
# Telegram
TELEGRAM_BOT_TOKEN=your_bot_token

# Database
POSTGRES_HOST=localhost
POSTGRES_PORT=5432
POSTGRES_DB=studobot
POSTGRES_USER=studobot
POSTGRES_PASSWORD=secure_password

# MinIO
MINIO_ENDPOINT=localhost:9000
MINIO_ACCESS_KEY=minioadmin
MINIO_SECRET_KEY=minioadmin
MINIO_BUCKET=student-files

# AI (опционально)
AI_API_KEY=your_ai_api_key
AI_MODEL=gpt-4

# n8n
N8N_HOST=localhost
N8N_PORT=5678
N8N_API_KEY=your_n8n_api_key
```
